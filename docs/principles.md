# Guiding Principles

Project RUN exists to preserve and make accessible the history of human creativity through computers and electronic media.

## 1. People are the subject

Hardware, software, magazines, disks, boxes, and cables matter because people used them. Catalog records should leave room for memory, motivation, community, labor, play, frustration, and discovery.

## 2. Objects are evidence of practice

An object is not only a model number or market value. Wear, modifications, handwritten notes, bundled media, packaging, and provenance can show how it was understood and used.

## 3. Preservation should lead to use

The project should not turn working culture into inert display pieces. Where responsible, materials should be tested, read, run, repaired, demonstrated, compared, and used to create new work.

## 4. Context is part of the object

Manuals, advertisements, reviews, magazine articles, catalogs, contemporary prices, software, recordings, and personal accounts help explain what an item meant in its own time.

## 5. Existing archives are partners

Project RUN should link to and build upon trustworthy public resources rather than duplicating them without reason. Internet Archive and similar collections can provide searchable digital surrogates while the physical copy supplies provenance and material evidence.

## 6. Accessibility is a preservation requirement

Knowledge should be available in portable formats and in multiple useful modalities. Records should be understandable by people, processable by machines, and suitable for future publication, search, audio, transcription, or visual presentation.

## 7. Open access is the direction

The purpose of collecting information is to learn from it and make it useful to others. Licensing decisions should favor reuse and public access wherever ownership and copyright permit.

## 8. Uncertainty should be visible

Incomplete knowledge is normal. Records should distinguish observation, memory, inference, and verified fact rather than hiding uncertainty behind false precision.

## 9. Start with useful records, not a perfect system

The metadata model should be consistent enough to search and migrate, but forgiving enough that cataloging can begin immediately. Missing fields are preferable to invented data.

## 10. The archive may create new things

Restoration, essays, exhibits, software tools, recordings, visual art, music, and experiments are not distractions from preservation. They are evidence that the material remains alive.
